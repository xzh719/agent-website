[ GLOSSARY · 07 ]

Reasoning · 推理范式

从"一问一答"到"深思熟虑"——让大模型像人类一样分步骤、多路径地思考问题。

CoT (Chain of Thought)

思维链

白话解释

思维链是一种提示技术：让模型在给出最终答案之前，先写下它的推理过程。就像数学老师要求你"写出解题步骤"而不是只写答案——把思考过程显式化，每一步都建立在前面步骤的基础上。

这句话听起来太简单了？但它确实有效，而且效果显著。在复杂推理任务上，加上一句"Let us think step by step"就能让准确率大幅提升。这不是因为模型变聪明了——

关键洞察：CoT 有效的根本原因是利用了自回归模型的特性——每个 token 的生成概率依赖于前面所有 token。当你让模型先写出推理步骤，这些步骤变成了后续 token 的"条件"，约束了概率分布，大幅降低了模型"跳到一个错误结论"的概率。CoT 不是让模型思考，而是用文本引导概率流向正确的方向。

CoT vs 直接回答

// 直接回答 (Zero-shot) 问: 房间有 3 人，每人 4 个苹果，每人吃 1 个，还剩多少？ 答: 9 个 ← 可能出错 // CoT 推理 问: 房间有 3 人，每人 4 个苹果，每人吃 1 个，还剩多少？ 答: [步骤1] 初始苹果 = 3 x 4 = 12 个 [步骤2] 吃掉 = 3 x 1 = 3 个 [步骤3] 剩余 = 12 - 3 = 9 个 答案: 9 个 ← 推理可审计、可纠错

ToT (Tree of Thoughts)

思维树

白话解释

思维树是思维链的升级版。CoT 走一条路，ToT 同时探索多条路。它把推理过程组织成一棵树：每个节点是一个"思考步骤"，从每个节点分支到多个可能的下一步，用模型评估每条路径的可行性，剪掉不好的分支，深入有希望的分支。

这本质上把推理变成了启发式搜索问题——就像下棋：在每一步考虑多种走法，评估优劣，选择最优。

// ToT 推理 (24 点游戏: 用 4, 6, 8, 12 算出 24) 第1层 (初始: 4, 6, 8, 12) ├── 分支A: 12 + 8 = 20, 剩余 {4, 6, 20} │ ├── 20 + 4 = 24 (成功!) │ └── 20 - 6 = 14 ... (剪枝) ├── 分支B: 12 - 4 = 8, 剩余 {6, 8, 8} │ └── ... (剪枝) └── 分支C: 6 x 4 = 24 (成功!) // BFS 广度搜索 + LLM 评分器做剪枝

ToT vs CoT：选哪个？

  • CoT：线性推理路径（数学题、逻辑推理）→ 一条路走到底
  • ToT：分支多、需探索（编程方案、创意生成、策略规划）→ 多路并进

ReAct (Reasoning + Acting)

推理-行动循环

白话解释

ReAct 是 Agent 的核心运行范式。它把一个复杂任务分解为多个"思考 → 行动 → 观察"的循环：

  1. Reason（推理）：分析当前状态，决定下一步做什么
  2. Act（行动）：执行决定的动作——调用工具、搜索、计算等
  3. Observe（观察）：接收行动的结果，判断任务是否完成
  4. 未完成 → 带着新观察回到第 1 步；完成 → 结束

ReAct 让 Agent 不再是"一次回答"，而是在与环境持续交互中逐步逼近目标——就像 PDCA 循环的 AI 版本。

// ReAct 循环伪代码 async function reactLoop(task, tools, maxSteps = 10) { let history = [{ role: "user", content: task }]; for (let step = 0; step < maxSteps; step++) { // REASON: LLM 分析并决定 const response = await llm.chat(history, tools); // 如果 LLM 输出最终答案，结束 if (response.finish_reason === "stop") break; // ACT: 执行工具调用 const result = await executeTool(response.tool_call); // OBSERVE: 将结果反馈给 LLM history.push(response.message); history.push({ role: "tool", content: result }); } return history; }

BFS 广度搜索 & 启发式搜索

BFS / Heuristic Search

白话解释

BFS（广度优先搜索）是图搜索算法：从起点开始，先探索所有"一步可达"的节点，再探索"两步可达"的节点，层层扩展。在 ToT 中，BFS 用于同时评估多个思维分支。

启发式搜索更进一步：用"启发函数"估算每个节点离目标有多远，优先探索最有希望的节点。在 ToT 中，LLM 自身充当了启发函数——给每个思考步骤打分，决定哪些分支值得深入。

关键洞察：ToT 把 LLM 从"生成器"变成"搜索算法中的评估器"。这是 Agent 架构 Brain 层的精髓——不是让 LLM 一次生成最终答案，而是让它参与结构化搜索过程，在每个决策点做判断。模型的推理能力被反复利用，每次只解决小问题（"这两个思路哪个更好？"），而非一次性解决整个大问题。
