Embedding · 向量嵌入

将语义"翻译"为数学——让计算机真正"理解"文字的含义，而非仅仅匹配关键词。

向量嵌入 (Embedding)

Vector Embedding / 嵌入向量

白话解释

向量嵌入是大模型理解语义的核心技术。简单说：把一段文本（一个词、一句话、一篇文章）转换成一串数字——通常是一个长度固定（如 1536）的浮点数数组。这段数字就是这段文本在高维空间中的"坐标"。

打个比方：想象一张巨大的地图，每段文字都在这张地图上有一个位置。意思相近的文字（如"猫"和"猫咪"）位置很近，意思无关的文字（如"猫"和"汽车"）位置很远。这张地图不是人画的——它是由神经网络在海量文本上训练出来的。训练的目标很简单：让经常出现在相似上下文中的词，拥有相似的坐标。

// 一段文字 → 一个高维向量 文本: "Agent 是一个智能系统" ↓ embed() 向量: [0.023, -0.451, 0.782, ..., 0.331] // 共 1536 个 float32 数字 // 这个向量编码了文本的语义 // 你可以用它来：搜索相似内容、分类、聚类、作为 LLM 输入

为什么不是关键词匹配？

传统的搜索用关键词匹配："咖啡店"只能搜到含"咖啡店"这三个字的文档。但向量嵌入搜索可以搜到"咖啡馆"、"coffee shop"、"星巴克"——因为它们在高维空间中距离很近。向量编码的是语义，而非字面。这就是为什么基于 embedding 的搜索远比关键词搜索智能。

Embedding 模型

嵌入模型

白话解释

Embedding 模型是专门用来把文本变成向量的模型。它和 ChatGPT 这种生成式模型不同——它不做对话，不写文章，唯一的任务就是：输入文本 → 输出向量。因为任务单一，它通常比生成式模型更小、更快、更便宜。

它是怎么训练出来的？

训练 embedding 模型不需要人工标注。核心思想叫对比学习（Contrastive Learning）：从互联网上抓取海量文本对——同一篇文章的两个段落、同一个问题的两种问法、同一张图片的文字描述——告诉模型"这两个应该近"。同时随机采样不相关的文本对，告诉模型"这两个应该远"。经过数亿次这样的训练，模型学会了把语义映射到空间位置。

关键洞察：Embedding 模型的本质是一个"语义压缩机"。它把任意长度的文本压缩成一个固定长度的向量，压缩的过程中保留了语义信息，丢弃了语法细节、修饰词等。一个好的 embedding 模型，压缩后的向量中，"猫"和"狗"的距离应该比"猫"和"飞机"近，"king - man + woman"应该约等于"queen"——这就是著名的词向量代数性质。

主流 Embedding 模型对比

模型 维度 特点 text-embedding-3-small 512 性价比最高，大规模检索首选 text-embedding-3-large 3072 精度最高，适合高精度匹配 Cohere Embed v3 1024 多语言优秀，中文表现好 BGE-M3 (BAAI 开源) 1024 本地部署，无需 API，完全免费 Jina Embeddings v2 768 支持 8K 长文本，适合长文档

浮点数向量 (Float32 Vector)

Float32Array · 高维向量

白话解释

当我们说"一个 1536 维的向量"，实际上就是一个包含 1536 个浮点数的数组，每个数字是 32 位浮点数（float32）。这 1536 个数字共同描述了文本在高维空间中的位置。

为什么要那么多维度？因为语义太复杂了。3 个维度只能描述"上下、左右、前后"，但语义有无数个层面：情感色彩、正式程度、主题类别、实体关系、时序信息……每增加一个维度，模型就多一个"角度"来区分文本。实践中发现 512-3072 维是一个甜蜜点——太少则区分度不够，太多则计算成本高且容易过拟合。

存储成本

// 一条 1536 维 float32 向量 1536 × 4 bytes = 6,144 bytes ≈ 6 KB // 100 万条向量 1,000,000 × 6 KB = 6 GB // 这就是为什么向量数据库需要专门的存储和索引策略
