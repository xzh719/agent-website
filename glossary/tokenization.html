[ GLOSSARY · 01 ]

Tokenization

理解大模型如何"阅读"文字——从人类语言到机器可计算的数字序列。

Token

词元 / 令牌

白话解释

Token 是大模型处理文本的最小单位。你可以把它理解为"模型认识的单词"，但它不完全是自然语言中的单词——一个 token 可能是一个完整的常用词（如 "the"），也可能是一个词的一部分（如 "play" + "ing" → 两个 token），或者一个标点符号，甚至是一个空格。

打个比方：如果说人类用"字"和"词"来阅读，那么大模型就用 token 来阅读。不同的是，模型的"词汇表"是经过数学优化的——它会把出现频率高的字符组合合并成一个 token，把罕见的组合拆开。

为什么重要

Token 数量直接决定了三件事：成本（API 按 token 计费）、速度（token 越多生成越慢）、上下文容量（每个模型有 token 上限）。中文通常比英文消耗更多 token，因为每个汉字常常被拆分为 1-3 个 token，而英文单词往往 1 个单词 = 1 个 token。这意味着同样的内容，中文输入的 API 费用可能比英文高 30-50%。

一个直观的例子

// 英文: "I love AI" → 3 tokens ["I", " love", " AI"] // 中文: "我爱人工智能" → 可能 5-7 tokens ["我", "爱", "人工", "智能"] 或 ["我", "爱", "人", "工", "智", "能"] // 这就是为什么同样的语义，中文 token 消耗更大

BPE (Byte Pair Encoding)

字节对编码

白话解释

BPE 是目前主流大模型（GPT 系列、Claude 等）构建 token 词表的核心算法。它的思路出奇地简单：从字符级开始，反复合并出现频率最高的相邻对，直到词表达到预设大小（比如 100,000 个 token）。

这个算法的巧妙之处在于——它不需要理解语言。它只是统计"哪两个东西经常挨在一起出现"，然后机械地把它们合并。结果却是：常用词被合并成单个 token（高效），罕见词保持拆分状态（灵活），新词可以通过已有子词拼出来（不会遇到"不认识的字"）。

工作原理 · 逐步推演

假设我们有一个迷你语料库，BPE 的训练过程如下：

// 初始状态：每个字符就是一个 token 语料: "low lower lowest" Step 1: 统计相邻对频率 l-o: 3次 o-w: 3次 w-空格: 1次 w-e: 2次 w-l: 1次 ... 最高频: l-o (3次) → 合并为 "lo" Step 2: 词表加入 "lo"，继续统计 lo-w: 3次 → 合并为 "low" Step 3: 词表加入 "low"，继续统计 low-e: 2次 → 合并为 "lowe" Step N: 不断迭代，最终词表包含: ["l", "o", "w", "e", "r", "s", "t", " ", "lo", "low", "er", "est", ...] // 实际 GPT 词表约 100,000 个 token，覆盖几十种语言
关键洞察：BPE 是一种"数据驱动"的压缩算法。它不依赖语言学规则——不需要词典、不需要语法知识。只靠统计频率，就能自动发现 "ing"、"tion"、"pre" 这些有意义的子词单元。这就是为什么同一个 BPE 词表可以同时处理英文、中文、代码、甚至 emoji——它只是统计字节对，不关心语义。

对使用者的实际影响

  • 英文常用词 ≈ 1 token/词，中文 ≈ 1.5-3 tokens/汉字 → 中文 API 成本更高
  • 代码中的缩进空格每个都是独立 token → 用 Tab 比空格更省 token
  • 专有名词、罕见字会被拆得更碎 → token 消耗更大
  • 不同模型的 tokenizer 不同 → 同样的文本在不同模型上的 token 数不一样

Tokenization

分词 / 词元化

白话解释

Tokenization 就是把原始文本转换成 token 序列的过程。它是大模型处理一切输入的第一道工序——在你向 ChatGPT 发送消息后、模型"理解"你的话之前，你的文字首先经过 tokenizer 被切成一个个 token，然后每个 token 被映射为词表中的数字 ID。

// Tokenization 的完整流程 输入文本: "Agent 架构解析" Step 1 · 预处理: 规范化 Unicode、处理特殊字符 → "Agent 架构解析" Step 2 · BPE 分词: 用训练好的词表切分 → ["Agent", " ", "架构", "解析"] // 注意："架构"和"解析"各自被合并为一个 token Step 3 · 映射为 ID: 每个 token 查表得到数字 → [1724, 220, 48621, 39105] Step 4 · 送入模型: ID 序列 → Embedding 层 → Transformer
关键洞察：模型从未"看到"过文字——它只看到数字。Tokenization 是自然语言和数学模型之间的翻译层。这层翻译的质量直接影响模型的理解能力：如果分词方式不合理（比如把"机器学习"拆成"机"+"器"+"学"+"习"而不是"机器"+"学习"），模型就需要消耗更多推理能力去"拼接"语义。
